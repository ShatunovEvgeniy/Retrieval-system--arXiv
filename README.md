# Retrieval-система по статьям arXiv

## Описание проекта
Разработка и оценка семантической поисковой системы (Retrieval) для научного корпуса статей arXiv. Система принимает текстовый запрос, находит наиболее релевантные публикации из датасета и возвращает ранжированный список ответов. Качество поиска оценивается метрикой Mean Reciprocal Rank (MRR@5), а производительность ключевых компонентов проходит детальное профилирование.

## Цель и достигнутые результаты
| Параметр | Значение |
|----------|----------|
| Целевая метрика | MRR@5 > 0.91 |
| Достигнутый результат | MRR@5 ≈ 0.93-0.95 |
| Среда выполнения | CPU (модели оптимизированы для инференса без GPU) |
| Профилирование | Выполнен покомпонентный замер времени (кодирование запроса, поиск в Qdrant, reranking, сортировка) |

## Датасет
| Файл | Структура | Описание |
|------|-----------|----------|
| arxiv-metadata-s.json | id, title, abstract, authors, categories, journal-ref, doi, update_date | Метаданные корпуса статей arXiv |
| test_sample.csv | query, id, abstract | Тестовые запросы с единственным верным ответом (id) |

## Архитектура и подход

Система реализована по принципу двухэтапного поиска (Retrieve & Rerank) с использованием векторной базы данных:

### Компоненты системы

1. **Векторная база данных (Qdrant с индексом HNSW)**
   - Хранит векторные представления статей
   - Индекс HNSW обеспечивает поиск за O(log n) вместо линейного перебора
   - Работает in-memory, легко масштабируется до production

2. **Модель первого этапа: Bi-Encoder (all-MiniLM-L6-v2)**
   - Преобразует текст запроса и документов в векторы размерности 384
   - Обеспечивает быстрый первичный отбор кандидатов (top-50)
   - Размер модели ~80 МБ, оптимизирована для CPU

3. **Модель второго этапа: Cross-Encoder (cross-encoder/ms-marco-MiniLM-L-6-v2)**
   - Принимает на вход пару (запрос, документ) и выдает скор релевантности
   - Переранжирует кандидатов из первого этапа для повышения точности
   - Обрабатывает только 50 документов на запрос, что минимизирует накладные расходы

4. **Логика оценки и профилирования**
   - Расчёт MRR@5 для оценки качества ранжирования
   - Замер времени выполнения каждого компонента системы
   - Визуализация распределения времени и позиций релевантных ответов

### Формирование индекса

Для векторизации используется конкатенация заголовка и аннотации:
```
search_text = title + " | " + abstract
```

В payload Qdrant хранятся метаданные для фильтрации и отображения:
- article_id, title, authors, categories, journal-ref, doi, update_date

### Схема работы

```
Запрос пользователя
        |
        v
Bi-Encoder: кодирование запроса в вектор
        |
        v
Qdrant (HNSW): поиск top-50 кандидатов по косинусному сходству
        |
        v
Cross-Encoder: переранжирование кандидатов по релевантности
        |
        v
Возврат top-5 article_id с метаданными
```

## Работа с проектом

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Подготовка данных
Поместите файлы `arxiv-metadata-s.json` и `test_sample.csv` в папку data, лежащую в корневой директорию репозитория.

Ссылка на данные: https://disk.yandex.ru/d/_hlESjxRVivrdg

### 3. Запуск
Откройте и последовательно выполните ячейки `retrieval_system.ipynb`. Все результаты расчётов, графики и выводы будут выведены в процессе выполнения.

## Конфигурация индекса Qdrant

```python
VectorParams(
    size=384, 
    distance=Distance.COSINE,
    hnsw_config=HnswConfig(
        m=16,              # количество связей в графе
        ef_construct=100,  # размер candidate pool при построении
        on_disk=False      # хранение в памяти для скорости
    )
)
```

## Итоговые выводы

Использование двухэтапной архитектуры с векторной базой данных Qdrant позволило достичь высокой точности семантического поиска, уверенно преодолев целевой порог MRR@5 > 0.91. Применение Bi-Encoder для быстрого первичного отбора и Cross-Encoder для точного ранжирования обеспечивает оптимальный баланс между скоростью и качеством выдачи.

Профилирование показало, что основная доля времени выполнения приходится на этап reranking, что является ожидаемым компромиссом для достижения высокой точности. Описанные в проекте архитектурные решения (индекс HNSW, in-memory режим Qdrant, модульная структура кода) формируют чёткий roadmap для перевода прототипа в production-среду с поддержкой масштабирования, фильтрации по метаданным и низкой латентностью.
