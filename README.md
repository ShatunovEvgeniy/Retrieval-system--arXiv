# Retrieval-система по статьям arXiv

## Описание проекта
Разработка и оценка семантической поисковой системы (Retrieval) для научного корпуса статей arXiv. Система принимает текстовый запрос, находит наиболее релевантные публикации из датасета и возвращает ранжированный список ответов. Качество поиска оценивается метрикой **Mean Reciprocal Rank (MRR@5)**, а производительность ключевых компонентов проходит детальное профилирование.

## Цель и достигнутые результаты
| Параметр | Значение |
|----------|----------|
| **Целевая метрика** | `MRR@5 > 0.91` |
| **Достигнутый результат** | `MRR@5 ≈ 0.92` |
| **Среда выполнения** | GPU T4 (модель `all-MiniLM-L6-v2`, ~22 МБ) |
| **Профилирование** | Выполнен покомпонентный замер времени (кодирование, поиск, сортировка) |

## Датасет
| Файл | Структура | Описание |
|------|-----------|----------|
| `arxiv-metadata-s.json` | `id`, `title`, `abstract` | Метаданные корпуса статей arXiv |
| `test_sample.csv` | `query`, `id`, `abstract` | Тестовые запросы с единственным верным ответом (`id`) |

## Архитектура и подход
Система реализована по принципу **Dense Retrieval**:
1. **Preprocessing** — конкатенация `title + " " + abstract`, заполнение пропусков, создание индексов `id ↔ index`.
2. **Embedding** — преобразование всех статей и запросов в векторное пространство с помощью `SentenceTransformer`.
3. **Search & Ranking** — вычисление косинусного сходства запроса со всем корпусом, выбор топ-5 через `torch.topk`.
4. **Evaluation & Profiling** — расчёт `MRR@5`, замер времени выполнения каждого этапа, визуализация распределения времени и позиций релевантных ответов.

## Работа с проектом
### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Подготовка данных
Поместите файлы `arxiv-metadata-s.json` и `test_sample.csv` в корневую директорию репозитория.
Ссылка на данные: ```https://disk.yandex.ru/d/_hlESjxRVivrdg```

### 3. Запуск
Откройте и последовательно выполните ячейки `retrieval_system.ipynb`. Все результаты расчётов, графики и выводы будут выведены в процессе выполнения.


## Итоговые выводы
Использование легковесной sentence-transformer модели позволило достичь высокой точности семантического поиска, уверенно преодолев целевой порог `MRR@5 > 0.91`. Система демонстрирует стабильное качество на академических запросах благодаря глубокому пониманию контекста и терминологии. Описанные в проекте оптимизации (FAISS, гибридный поиск, квантизация) формируют чёткий roadmap для перевода прототипа в production с низкой латентностью и высокой масштабируемостью.
